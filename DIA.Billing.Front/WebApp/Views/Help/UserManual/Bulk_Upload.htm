Bulk Upload

 

  1. Select the BU and Product Category (Default value for Product Category is 'CCBS' and for BU is ‘All’)

 

Special Note

  • BU option, ‘All’ is not allowed for Bulk Upload option.

 

  1. Select the ‘Bulk Upload’ radio button and click on 'Upload' button.
  2. System will pop up a new window (Bulk Upload GUI)
  3. Select ‘Choose File’ button.
  4. A new window will be opened and using that user will be able to select the relevant file.
  5. Then needs to click on 'Add' button, upon clicking on Add button all the details in the excel will be loaded to the grid.

 

Special Note

  • Excel sheet will be created according to a predefined format.

 

ConnectionReference Amount Remarks ContractNumber ReferenceNumber

  23.45   60714800  

 

 

  1. Then user have to click on ‘Verify’ button.
  2. All the details will be verified and verification passed details will be loaded to the ‘Valid Numbers’ grid and verification failed numbers will be loaded to the ‘Invalid Numbers’ grid. Additionally, details in the 'Upload Summary' will be updated with respect to the data in the excel.
  3. User will be able to edit the invalid numbers and correct them and then pass them to ‘Valid numbers’ grid by clicking on ‘Apply’ button.
  4. Additionally, without correcting invalid numbers, user can submit them to a suspense account by clicking on ‘Change to suspense’ button.
  5. Upon clicking this, Connection reference / Contract ID of the invalid number will be changed to their respective suspense account number's connection reference and the original connection reference / contract ID will be added to the ‘remarks’ field along with its’ already existing remarks details.
  6. Users will click on submit button. Upon clicking, all the records will be passed to bulk payment entry GUI’s grid and will show those data along with the related other customer details such as Customer Name, Customer ID, BU & etc.
  7. ‘Upload summary’ details such as ‘Transfer to suspense count’ and ‘Transfer to suspense amount’ will be shown under ‘batch summary’.
  8. If user wants he/she can retrieve more details from RBM, PE, OCS and Other systems by clicking on respective line-wise button (to retrieve details of one number) or bulk buttons (to retrieve details for all the numbers).

  9. If user wants he/she will be able to attach a file or add remarks to that particular batch he/she is creating.

 

Special Note:

  • If user adds a remark to the Remark field in ‘batch summary’ widget, then that same remark will be reflected in the Remark column in the grid. However if user wants to add a custom remark to one record he/she will be able to edit that respective cell and add a custom remark.

 

  1. Then user has to select the payment mode from the dropdown

  2. Finally user will click on ‘save’ button to submit the data

  3. Upon saving receipt numbers will be generated and saved details will be loaded to

  4. Receipt Details Tab

  5. If necessary user can print receipt individually or bulk-wise
