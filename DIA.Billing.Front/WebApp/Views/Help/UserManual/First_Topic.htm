Random Bill Payment Radio Option

 

  1. Select the  BU and Product Category (Default value for Product Category is 'CCBS' and for BU is ‘All’)
  2. Select the ‘Random Bill Payment’ radio button
  3. Then a separate pop-up will be opened. This pop-up will have a grid with four columns (Connection Reference, Contract Number, Amount and Reference Number).
  4. User can key-in connection reference (or Contract Number), Pay amount and Reference Number (Cargils/Keels receipt No) to the relevant cells in the grid. Upon pressing enter key, cursor will move across the cells. However cursor will not move to ‘Reference Number’ cell. User will have to manually point the cursor to that cell if they want.
  5. Upon keying in records 'No of record' field and 'Total Amount' field will update automatically
  6. Finally user have to click on submit button to pass keyed-in data to the Payment Entry GUI

 

Special Note:

  • Always red color row should not contain any data. Data in the red color raw will not be submitted to the Payment Entry GUI.

 

  1. Upon submitting, connection reference/contract number will be verified and if verification is success additional details such as customer name, BU, Customer ID and etc will be shown in the main grid.

 

Special Notes:

  • If user has keyed-in an invalid number, then those will be hi-lighted and shown. User can edit the number and validate it again by clicking on CRM button.

  • If user has keyed-in a pre-paid number, upon validating that particular number will be highlighted and shown. System will not facilitate taking payments to prepaid numbers.

 

  1. If user wants they can retrieve more details from RBM, PE, OCS and Other systems by clicking on respective line-wise button (to retrieve details of one number) or bulk buttons (to retrieve details for all the numbers).

  2. If user wants he/she will be able to attach a file or add remarks to that particular batch he/she is creating.

 

Special Notes:

  • If user adds a remark to the Remark field in ‘batch summary’ widget, then that same remark will be reflected in the Remark column in the grid. However if user wants to add a custom remark to one record he/she will be able to edit that respective cell and add a custom remark.

 

  1. Then user can select the relevant records he/she wants to submit

  2. Then user has to select the payment mode from the dropdown

  3. Finally user will click on ‘save’ button to submit the data

  4. Upon saving receipt numbers will be generated and saved details will be loaded to Receipt Details Tab

  5. If necessary user can print receipt individually or bulk-wise
