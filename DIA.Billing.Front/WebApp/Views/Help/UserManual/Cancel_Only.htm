Cancel Only

  1. Find the receipt/s details which needs to cancel.
    • User can search the relevant receipt/s using ‘Bulk Payment’ UI or ‘Payment Inquiry’ UI.

 

Special Note

    • Frontline users will not be able to initiate a cancellation via Payment Inquiry UI. Only BAckOffice users will be able to initiate a cancellation via Payment Inquiry UI.

 

    • If user wants cancel a single receipt, then line-wise cancel button needs to click.
    • If user wants to cancel multiple receipts then user has to tick on the relevant check boxes and then click on 'Cancel Batch' button.

 

  1. Relevant receipt details will be loaded to the 'Cancel Receipt Details' grid in the Payment Cancellation GUI.
  2. Select the ‘Cancellation Type’ as ‘cancel only’. Upon selecting this, ‘Correct Entry Details’ grid will be hide.
  3. Select / key-in details such as Cancellation Reason code / description, Mistake done by and cancellation remarks.
  4. If necessary, attach file to that particular batch and select the types of files that are attaching.

 

Special Note

  • For some users attaching documents is compulsory. Refer following diagram for more info.

  1. Select the payments that needs to cancel.

Special Note

  • If the cancellation is submitted by the frontline users, then cancellation request will be submitted to BackOffice user’s approval. Workflow Ref will be generated along with a Cancellation Batch ID.
